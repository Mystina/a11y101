    Hard Copy Prints Assets

    • ePub

    • Use heading levels correctly with always h1 as the first heading level in each page, section, and region
    • When use table markups, ensure screen readers can navigate elements within the table, such as column header information and all cells. Having "live" tables allows for font size increase and ability to tab from cell to cell on some readers
      • Include use of thead to group multi-row headings
      • th required to indicate headings and subheadings
    • When use list markups( ol, ul), ensure screen readers identify hierarchy of the list and its sublists. Also enable dynamic list numbering
    • em tags used to indicate all text that is to be vocally stressed
    • strong tags used to indicate where importance is being conveyed
    • i and b tags used for non-emphasized text that is semantically significant and indicates a change in tone
    • CSS styling (span class=”bold”) for presentational bolding and italics need to be recognized by screen readers
    • HTML documents should have a language assigned
    • All linked content is also accessible to the reader
    • Captions and titles where available, marked up appropriately using HTML5 (figure/figcaption
    • Alt text provided for all significant images, where proper description is lacking for a non-sighted reader
      • Do not repeat the figure caption or any content directly from the surrounding body paragraphs
      • The vendor will not make changes to the figure or table captions. Suggestions on changes to figure captions can be made by Kaplan at the time of review
      • The vendor will tailor alt text descriptions to be at the level and vocabulary of the intended audience, i.e. SAT high school student versus Medical student
      • Keep to as close to 140 characters
      • The screen reader software announces "graphic" then the alt text. Do not add "An image of" to the alt text
    • Alt attribute left empty for decorative images, including background images which should not convey information
    • Use tables for conveying tabular information not for presentation
    • MathML markup included for all math content
      • All math expressions and/or equations must be entered as MathML (in an equation editor such as the one included with MS Word, Habitat or in Google docs)
      • Fractions and exponents are represented using MathML
      • Plain numbers or isolated variables okay as text (e.g. 7 or x)
      • Single variables are okay as inline text= Ex. x + 5 =
      • Simple equations are okay as inline text if no unicode: Ex. 1+2
      • Alt attribute left empty for decorative images, including background images which should not convey information. If unicode symbols is present the entire equation should be represented instead as MathML, Ex. x ≠−5
      • Span class=”equation” not read by screen reader and not useful for this purpose
    • Mathjax rendering code has been added/enabled
